### 8. 最终方案：设备级回归 + 事件级回归

#### 8.1 设备级回归（主力模型）

每台设备一行，目标=平均MTBF（小时数），连续数值回归。

**特征工程（11维）**

| 特征 | 含义 |
|------|------|
| `n_failures` | 设备总故障次数 |
| `std_mtbf_h` | 故障间隔标准差 |
| `min_mtbf_h` / `max_mtbf_h` | 最短/最长故障间隔 |
| `avg_repair_h` / `max_repair_h` | 平均/最长维修时长 |
| `aging_days` | 设备运行天数 |
| `fail_rate_monthly` | 月均故障率 = n_failures/aging_days×30 |
| `station_enc` / `brand_enc` / `line_enc` | 车站/品牌/线路编码 |

**关键处理**

- **过滤**：≥5次故障 + 去除<10h和>P95极端间隔（1523台设备）
- **分层划分**：按MTBF分5档 → 设备级`stratify`划分80/20
- **log变换**：`np.log1p(y)` → 训练 → `np.expm1(pred)` 还原
- **SMOTE**：少数类过采样到均衡

**模型结果（305台设备测试集）**

| 模型 | MAE | MAPE | **1-MAPE** | R² | 训练时间 |
|------|:---:|:---:|:---------:|:---:|:---:|
| **★ MLP(PyTorch)** | 3.5天 | 13.1% | **86.9%** | 0.892 | 49s |
| RF | 3.8天 | 13.5% | 86.5% | 0.867 | 4s |
| LightGBM | 3.8天 | 13.6% | 86.4% | 0.869 | 1s |
| XGBoost | 3.9天 | 14.1% | 85.9% | 0.856 | 2s |

**MLP架构：** 输入(11) → 128 → 64 → 32 → 输出(1)，BatchNorm + ReLU + Dropout(0.3)，HuberLoss，AdamW优化器，ReduceLROnPlateau调度

#### 8.2 事件级回归（辅助模型）

每次故障一行，目标=单次故障间隔（小时数），预测下一次故障。

**Top10特征（经特征筛选）：** hour, month, repair_dur, response, n_hist, last_interval, avg_interval, aging_days, fail_rate, station_enc

**模型：** RF300, max_depth=20, min_samples_leaf=2

**结果： 1-MAPE = 72.0%**（MAE=64.0h, RMSE=124.6h）

#### 8.3 设备级 vs 事件级

| | 设备级 ★ | 事件级 |
|---|---|---|
| 粒度 | 1523台设备 | 13339次故障 |
| 目标变量 | 平均MTBF | 单次故障间隔 |
| 1-MAPE | **86.9%** | 72.0% |
| 适用场景 | 周期性检修排班 | 单次故障应急参考 |
| 模型 | MLP(PyTorch) | RF300 |

事件级精度低于设备级，原因是单次故障间隔自身随机性大（同一设备波动157%），均值操作是消除这种随机性的唯一方式。

#### 8.4 分布式训练探索

尝试了三种方案在Spark集群上分布式训练：

| 方案 | 实现 | 精度(1-MAPE) | 时间 | 结论 |
|------|------|:----------:|:----:|------|
| MLlib RF/GBT | Spark原生 | 52.1% | 277s+811s | 分桶近似损失精度 |
| MLlib优化版 | maxBins+subsampling调优 | 52.1% | 58s+114s | 速度提升5-7×,精度无改善 |
| **numpy手写RF** ★ | 纯Python+numpy,精确分裂 | **86.2%** | 169s | 精度接近sklearn,慢42× |

**核心发现：**

| | sklearn | numpy手写 | MLlib |
|---|---|---|---|
| 分裂算法 | 精确分裂点 | 精确分裂点 | 分桶近似(maxBins) |
| 实现语言 | Cython编译 | Python+numpy | JVM |
| 精度 | 86.5% | 86.2% | 52.1% |
| 速度 | 4s | 169s(42×) | 277s |

**结论：** 算法层面,手写RF和sklearn等价(同为精确分裂点+Bagging),精度几乎一致(差0.3%)。MLlib精度差的根因是分桶机制在小数据上信息损失过大,与分布式/单机无关。手写版比sklearn慢42倍在于Python解释执行 vs Cython编译——循环仍由解释器处理,仅内部运算靠numpy加速。
